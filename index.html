Skill 01 Text · any model · zero setup

Prompt Optimizer

A better prompt, plus the receipts.

Hundreds of millions of people write prompts now — and most write bad ones. Existing rewriters change wording but never prove they helped, and a prompt tuned for one model silently underperforms on another. Prompt Optimizer diagnoses the weaknesses, rewrites with the right techniques, tunes per model, generates test cases, and renders a BEFORE/AFTER so you can see the lift.

Before / after

Same prompt — a support bot grounded in help docs. Asked a question the docs don't cover.
Before raw prompt → model
"answer the customer's question using our help docs"
  • Invents a 30-day refund policy that doesn't exist — no grounding instruction.
  • Docs + question concatenated → injection risk.
  • Free-form length & tone, no escalation path.
  • Switch to Claude later → quality shifts, no portability.
  • No tests, no evidence — you find failures in production.
After with Prompt Optimizer
// GPT variant (excerpt) Answer using ONLY the HELP_DOCS below. ### HELP_DOCS {docs} ### CUSTOMER_QUESTION {question} If not in HELP_DOCS, reply exactly: "…routing you to a human." escalate=true Output JSON: {answer, source_quote, escalate}
  • Grounded + quotes its source; escalates instead of guessing.
  • Docs delimited from the question — injection-safe.
  • Ships a 4-case test set (happy · not-in-docs · injected · empty).
  • Claude / DeepSeek variants included — portable.
On the not-in-docs test: Before invents a fake policy · After returns {"escalate": true} and nothing invented. From “trust me” to a provable, reusable lift.
◆ Model-aware: GPT · Claude · Gemini · DeepSeek · Grok · GLM ⛔ Safety-gated — refuses abuse prompts ✚ Injection-resistant — payload is content, not commands ≡ Light Mode — won't over-engineer a trivial prompt ✳ Redacts secrets & API keys

Skill 02 Visual · Lumi + Sonic + code sandbox

Data Storyteller

Messy numbers in, a true story out.

Turning a spreadsheet into a shareable, on-brand visual is a 2–14 hour, multi-tool grind (Excel → screenshot → Canva → a video tool) — and generic AI tools hand back off-brand drafts that sometimes draw the trend the wrong way. Data Storyteller does the whole chain in one governed pass, and its single obsession is that the story is true: every figure is traced to a source cell or computed in a sandbox before a pixel is drawn.
